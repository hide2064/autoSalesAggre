# 設計ドキュメント: 売上集計自動化ツール (autoSalesAggre)

## 概要

複数のタブ区切りTSVファイルを一括読み込みし、ヘッダーが不揃いなデータを「allシート」へ統合する。Configシートのマスタ情報を参照して付加情報を算出・集計し、部署別・期間別の動的分析を可能にするExcel VBAツール。

---

## シート構成

### 固定シート

| シート名 | 役割 |
|---------|------|
| main | ファイル選択ボタン、実行ログ（A列タイムスタンプ＋B列メッセージ） |
| Config | 製品マスタ・口銭マスタ・ヘッダー名寄せ・部署リストの4テーブル |
| all | 全データ統合シート（ソース列＋計算2列） |
| 集計 | 部署・期間フィルタ＋階層集計テーブル |

### 動的シート

読み込んだTSVファイル名をそのままシート名として作成する生データシート。同名ファイル再読み込み時は既存シートを削除して再作成する。

---

## allシート 列定義

| # | 列名 | 内容 |
|---|------|------|
| 1 | 客先名 | ソースデータ |
| 2 | 製品コード | ソースデータ |
| 3 | 売上金額 | ソースデータ |
| 4 | 製品単価 | ソースデータ |
| 5 | 売上数量 | ソースデータ |
| 6 | 売上発生日 | ソースデータ |
| 7 | 売上種別 | ソースデータ |
| 8 | 部署 | ソースデータ |
| 9 | 製品名 | 製品コード→製品マスタ参照（計算列） |
| 10 | 部署取り分 | 売上金額×口銭比率/100（計算列） |
| 11 | ソースファイル | 読み込み元ファイル名（デバッグ用） |

全列は文字列として読み込み、先頭ゼロを保持する。売上金額・製品単価・売上数量・売上発生日・部署取り分はallシート転記後に適切な型へ変換する。

---

## Configシート レイアウト

```
[A1] 製品マスタ
[A2] 製品コード  [B2] 製品名
[A3以降] データ行

[D1] 口銭マスタ
[D2] 売上種別   [E2] 口銭比率%
[D3以降] データ行

[G1] ヘッダー名寄せ設定
[G2] 正規名     [H2] 対応列名（カンマ区切り）
[G3以降] データ行
  例: G3=売上金額, H3=金額,Amount,売上高

[J1] 集計用部署リスト
[J2] 全部署（固定）
[J3以降] ユニーク部署名
```

部署リスト（J列）はallシート再構築後に自動更新し、集計シートB2のプルダウン（入力規則）を連動して再設定する。

## モジュール構成

| モジュール | 主な責務 |
|-----------|---------|
| `modConfig` | Configシートから各マスタをDictionaryへ読み込む。部署リスト更新。 |
| `modFileIO` | ファイル選択ダイアログ・TSV読み込み・生データシート展開 |
| `modDataProcess` | allシート初期化・各生データシートからの転記・計算列付与 |
| `modAggregation` | 集計条件読み取り・Dictionary集計・テーブル描画 |
| `modUIControl` | ボタンイベント・ログ書き込み・Application.ScreenUpdating制御 |

---

## 処理フロー

```
[ユーザー] mainシートのボタンクリック
    │
    ▼
modUIControl.RunAll()
    ├─ modConfig.LoadAll()             ← Dictionary×3をメモリ展開
    ├─ modFileIO.SelectAndLoad()       ← ファイル選択→生データシート作成
    ├─ modDataProcess.BuildAllSheet()  ← allシート統合・計算列付与
    └─ modConfig.RefreshDeptList()     ← 部署リスト更新→集計B1プルダウン再設定

[ユーザー] 集計シートB2（部署）またはD2/D3（期間）変更
    │
    ▼
Worksheet_Change イベント
    └─ modAggregation.Rebuild()        ← allシートから条件抽出→集計テーブル描画
```

## Dictionary 設計

| 変数名 | Key | Value | 生成元 |
|--------|-----|-------|--------|
| `dictProduct` | 製品コード（String） | 製品名（String） | 製品マスタ |
| `dictCommission` | 売上種別（String） | 口銭比率（Double） | 口銭マスタ |
| `dictHeaderMap` | ソース列名（小文字・トリム正規化） | 正規列名（String） | 名寄せ設定 |

集計処理では `dictSummary` を使用:

- Key: `製品名 & "||" & 客先名`（区切り文字 `||` で衝突を回避）
- Value: `Array(売上金額合計, 数量合計, 口銭合計)` の Variant配列

描画時は `dictSummary.Keys` を取得後、製品名（`||` 前部分）でバブルソートまたは配列ソートを行い、製品名グループが連続するよう並べてからセル転記する。

## 集計シート UI・出力形式

### レイアウト

```
  A         B             C             D
  ─────────────────────────────────────────
1 部署選択  [全部署▼]
2 開始日    [2024/01/01]
3 終了日    [2024/12/31]
4
5           売上金額合計  売上数量合計  口銭総額
6 製品Aアルファ  1,200,000    150       120,000   ← 製品行（親）
7   得意先X社     800,000    100        80,000   ← 客先行（子）
8   得意先Y社     400,000     50        40,000
9 製品Bベータ     600,000     80        60,000
10  得意先Z社     600,000     80        60,000
11 ─────────────────────────────────────────
12 総合計       1,800,000    230       180,000
```

### 描画ルール

- **製品行（親）**: 背景色あり（薄いグレー `RGB(220,220,220)`）、太字
- **客先行（子）**: 背景色なし、A列を全角スペース2文字インデント
- **総合計行**: 上罫線＋太字
- 集計テーブルは毎回全行削除→再描画（既存行上書き不可）
- `Worksheet_Change` のトリガー対象: B1（部署）、B2（開始日）、B3（終了日）

### フィルタ条件の組み合わせ

| 部署選択 | 期間 | 動作 |
|---------|------|------|
| 全部署 | 指定あり | 期間のみ絞り込み |
| 特定部署 | 指定あり | 部署＋期間で絞り込み |
| 特定部署 | 空欄 | 部署のみ絞り込み |
| 全部署 | 空欄 | 全データ集計 |

## エラーハンドリング

| ケース | 対応 |
|--------|------|
| 製品コードがマスタ未登録 | allシートの製品名列に `[未登録]` を記入、mainログに警告 |
| 売上種別がマスタ未登録 | 部署取り分を `0` とし、mainログに警告 |
| ファイル読み込みエラー | 該当ファイルをスキップ、mainログにエラー記録、処理継続 |
| ファイル選択キャンセル | 即時終了（メッセージなし、正常終了扱い） |
| 集計シート期間入力が不正 | メッセージボックスで通知し、集計を中断 |

---

## 実装上の制約・方針

- セルへの直接書き込みループを避け、Variant配列でメモリ処理後に一括転記（`Range.Value = arr`）
- `Application.ScreenUpdating = False` / `Application.Calculation = xlCalculationManual` を処理前後で制御
- マジックナンバーは `modConfig` 内の定数（`Const COL_PRODUCT_CODE As Integer = 2` 等）で管理
- Configシートの各テーブル開始セルも定数で定義し、レイアウト変更への耐性を持たせる
- Microsoft Scripting Runtime（Dictionary）を使用するため、参照設定で `Microsoft Scripting Runtime` を有効にする

---

## 対象環境

- Excel 2016 以降（Windows）
- VBA 7.x
- Microsoft Scripting Runtime 参照設定必須
