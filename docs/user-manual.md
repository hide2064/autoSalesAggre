## 1. 概要

`autoSalesAggre.xlsm` は、複数の部署や担当者から収集した売上TSVファイルを自動的に統合・集計するExcelマクロブックです。

**主な機能:**

- 列名が異なる複数TSVを、Config設定で自動名寄せして統合
- 製品マスタ・口銭マスタから製品名・部署取り分を自動付与
- 集計シートで部署・日付を絞り込んで階層集計を即時表示

---

## 2. 初回セットアップ

### 2-1. Excel の設定

マクロブックを生成する前に、一度だけ行う設定です。

1. Excelを起動する
2. メニューから **ファイル → オプション** を開く
3. **トラストセンター → トラストセンターの設定** をクリック
4. **マクロの設定** を選択
5. **「VBAプロジェクト オブジェクト モデルへのアクセスを信頼する」** にチェックを入れる
6. OKで閉じる

> この設定はワークブック生成時（VBScript実行時）のみ必要です。通常運用では不要です。

### 2-2. ワークブックの生成

コマンドプロンプト（またはPowerShell）を開き、プロジェクトフォルダに移動して以下を実行します:

```
cscript setup\create_workbook.vbs
```

実行後、以下のメッセージが表示されれば成功です:

```
Setup started
Source : C:\...\src\
Output : C:\...\autoSalesAggre.xlsm
Importing VBA modules...
Modules imported.
Running InitWorkbook...
InitWorkbook complete.
Saved: C:\...\autoSalesAggre.xlsm
Done. Open autoSalesAggre.xlsm in Excel and enable macros.
```

### 2-3. ワークブックを開く

1. `autoSalesAggre.xlsm` をExcelで開く
2. 上部の **「マクロを有効にする」** ボタンをクリックする（セキュリティ警告が出た場合）

## 3. Configシートの設定

「Config」シートには、ツールが参照するマスタデータを入力します。初回は以下のサンプルデータが入力されています。実際の運用データに合わせて編集してください。

### 3-1. 製品マスタ（A列・B列）

売上データの「製品コード」から「製品名」を解決するためのマスタです。

| A列（製品コード） | B列（製品名） |
|----------------|------------|
| P001 | 製品A |
| P002 | 製品B |
| ... | ... |

**入力ルール:**
- A列に製品コード、B列に製品名を1行1件で入力
- 空行があると、そこで読み込みが終了するため、間に空行を入れないこと
- 同じ製品コードが複数ある場合は最初の行が優先される

### 3-2. 口銭マスタ（D列・E列）

「売上種別」ごとの口銭率（%）を定義します。口銭額は `売上金額 × 口銭率 / 100` で計算されます。

| D列（売上種別） | E列（口銭比率%） |
|--------------|---------------|
| 直販 | 10 |
| 代理店 | 5 |
| ... | ... |

**入力ルール:**
- D列に売上種別、E列に数値（%）を入力
- E列は必ず数値で入力すること（文字列の場合は0として扱われる）
- マスタに存在しない売上種別は口銭額が0になる

### 3-3. ヘッダー名寄せ設定（G列・H列）

読み込むTSVファイルの列名が異なる場合に、正規名に統一するための設定です。

| G列（正規名） | H列（対応列名・カンマ区切り） |
|------------|--------------------------|
| 客先名 | 得意先名,得意先コード,顧客名 |
| 製品コード | 品番,ProductCode |
| 売上金額 | 金額,Amount,売上高 |
| ... | ... |

**入力ルール:**
- G列に正規名（allシートの列名）、H列にTSVで使われている列名をカンマ区切りで入力
- スペースは無視されるため、`得意先名, 顧客名` のようにスペースを入れても可
- 同じエイリアスが複数の正規名に登録された場合は最初の登録が優先される
- 正規名そのものも自動的に一致対象になるため、完全一致する列名はエイリアス登録不要

**初期設定済みの正規名:**

| 正規名 | 意味 |
|-------|------|
| 客先名 | 売上先の顧客・得意先名 |
| 製品コード | 製品を識別するコード |
| 売上金額 | その行の売上金額 |
| 製品単価 | 1個あたりの単価 |
| 売上数量 | 販売数量 |
| 売上発生日 | 売上が発生した日付 |
| 売上種別 | 直販・代理店などの区分 |
| 部署 | 担当部署名 |

### 3-4. 集計用部署リスト（J列）

TSVファイル読み込み後に自動更新されます。**手動編集は不要です。**

## 4. TSVファイルの読み込み（RunAll）

### 4-1. 手順

1. **main シート**を開く
2. **「ファイルを読み込む」**ボタンをクリックする
3. ファイル選択ダイアログが開くので、読み込むTSVファイルを選択する
   - 複数ファイルを同時に選択可能（Ctrl+クリック または Shift+クリック）
   - 対応形式: `.txt` / `.tsv` / `.csv`（タブ区切り形式）
4. 「開く」をクリックする
5. 処理が実行され、main シートのログに進捗が表示される
6. 完了後、「all」シートにデータが統合されている

### 4-2. 処理の流れ

```
① ファイル選択ダイアログ
  ↓ ファイルごとに繰り返し
② TSVを「all」シート以外の一時シートに読み込む
  （シート名 = ファイル名から自動生成）
  ↓
③ ヘッダー行を名寄せ設定でマッピング
  ↓
④ 製品マスタで製品名を解決
  ↓
⑤ 口銭マスタで部署取り分を計算
  ↓
⑥ 「all」シートに追記
  ↓
⑦ 一時シートを削除
  ↓
⑧ 集計用部署リストを更新
  ↓
⑨ 集計シートで自動集計を実行
```

### 4-3. 実行ログの見方

main シートのログには各ステップの状況が記録されます。

| ログ例 | 意味 |
|--------|------|
| `RunAll started` | 処理開始 |
| `Loading: 売上データ.txt` | そのファイルの読み込み開始 |
| `Loaded N rows from 売上データ.txt` | 読み込んだ行数 |
| `BuildAllSheet complete. N rows written.` | all シートへの書き込み完了 |
| `RunAll complete.` | 全処理完了 |

### 4-4. TSVファイルの要件

- ファイルはタブ区切りテキスト形式（.txt / .tsv / .csv）
- 1行目がヘッダー行であること
- 文字コードはShift-JIS（Excel標準）を推奨
- 必須列: 客先名・製品コード・売上金額・売上発生日・部署（これら以外は空欄でも可）
- 列の順序は任意（名寄せ設定で対応）

## 5. 集計シートの使い方

### 5-1. フィルタの設定

「集計」シートを開くと、上部にフィルタ入力欄があります。

```
部署選択  [全部署 ▼]     ← B1: ドロップダウンで選択
開始日    [          ]   ← B2: 日付を入力（例: 2026/01/01）
終了日    [          ]   ← B3: 日付を入力（例: 2026/03/31）
```

**部署選択（B1）:**  
ドロップダウンリストから対象部署を選択します。  
「全部署」を選択すると全部署のデータが集計されます。

**開始日・終了日（B2・B3）:**  
日付を入力すると、指定期間内の売上データのみ集計されます。  
空白にすると日付フィルタは適用されません（一方だけ入力も可）。

> フィルタを変更すると **自動的に集計が再実行**されます。ボタン操作は不要です。

### 5-2. 集計結果の読み方

集計表は製品名を親として、客先名を子とした階層構造で表示されます。

```
製品A          1,200,000  120  120,000   ← 製品小計（太字・灰色背景）
  得意先X社      800,000   80   80,000   ← 客先別（全角スペースでインデント）
  得意先Y社      400,000   40   40,000
製品B            500,000   50   25,000
  得意先Z社      500,000   50   25,000
総合計         1,700,000  170  145,000   ← 全体合計（太字・上罫線）
```

| 列 | 内容 |
|----|------|
| A列 | 製品名（親行）または客先名（子行、全角スペース2文字インデント） |
| B列 | 売上金額合計 |
| C列 | 売上数量合計 |
| D列 | 口銭総額（部署取り分） |

### 5-3. フィルタのリセット

- 部署フィルタをリセット: B1 を「全部署」に変更
- 日付フィルタをリセット: B2・B3 の内容を削除（Delete キー）

## 6. ヘッダー名寄せの設定

部署や担当者ごとにTSVの列名が異なる場合、Config シートのヘッダー名寄せ設定でまとめて対応できます。

### 例: 新しい列名のエイリアスを追加する

ある部署のファイルで「顧客コード」という列名が使われている場合:

1. Config シートを開く
2. G列の「客先名」行（G3）に対応するH列（H3）を確認する  
   現在: `得意先名,得意先コード,顧客名`
3. H3 の末尾に `,顧客コード` を追加する  
   更新後: `得意先名,得意先コード,顧客名,顧客コード`
4. 次回のRunAll時から反映される

### マッピングされなかった列の扱い

TSVの列名がどの正規名にもマッピングされなかった場合、その列のデータはallシートに取り込まれません（無視されます）。

## 7. 運用上の注意事項

### データの更新について

- **RunAll は毎回 all シートを全件再生成**します（前回のデータは削除されます）
- 定期集計の場合は、毎回同じTSVファイルセットを選択し直してください
- all シートのデータを直接編集しないでください（次回RunAllで上書きされます）

### ファイル名とシート名

- RunAll実行時、各TSVは一時的に専用シートに読み込まれ、処理後に自動削除されます
- ファイル名は自動的にシート名に変換されます（一部の記号は除去、31文字に切り詰め）

### src フォルダ変更後の再構築

src フォルダ内の .bas ファイルを修正した場合:

1. `autoSalesAggre.xlsm` を削除する
2. `cscript setup\create_workbook.vbs` を再実行する

### バックアップ

- all シートのデータは処理のたびに削除・再生成されるため、必要なら別ファイルに保存してください
- Config シートのマスタデータは手動編集内容が保存されるため、定期的にバックアップすることを推奨します

## 8. エラーと対処方法

### 「開始日の形式が正しくありません」

**原因:** 集計シートのB2に日付として認識できない値が入力されている  
**対処:** B2の内容を削除するか、`2026/01/01` 形式で再入力してください

### 「終了日の形式が正しくありません」

**原因:** 集計シートのB3に日付として認識できない値が入力されている  
**対処:** B3の内容を削除するか、`2026/12/31` 形式で再入力してください

### ファイル選択後に何も起きない

**原因:** ファイルの形式がタブ区切りでない可能性がある  
**対処:** ファイルをテキストエディタで開き、列の区切り文字がタブであることを確認してください

### 製品名が空欄になる（all シート I列）

**原因:** TSVの製品コードがConfig製品マスタに登録されていない  
**対処:** Config シートのA列・B列に該当の製品コードと製品名を追加してください

### 口銭総額が 0 になる（all シート J列）

**原因1:** TSVの売上種別がConfig口銭マスタに登録されていない  
**対処1:** Config シートのD列・E列に該当の売上種別と口銭率を追加してください

**原因2:** Config口銭マスタのE列が数値ではない（文字列として入力されている）  
**対処2:** E列のセルを数値形式で再入力してください

### 集計シートが空のまま

**原因:** all シートにデータがない、またはフィルタ条件に一致するデータがない  
**対処:** まず「ファイルを読み込む」ボタンでデータを読み込んでください。データがある場合はフィルタ条件（部署・日付）を確認してください

### VBScript実行時エラー（セットアップ時）

**原因:** Excel のトラストセンター設定が未完了  
**対処:** [2-1. Excel の設定](#2-1-excel-の設定) を確認し、「VBAプロジェクト オブジェクト モデルへのアクセスを信頼する」にチェックが入っていることを確認してください

### マクロが無効になっている

**原因:** Excelのセキュリティ設定でマクロがブロックされている  
**対処:** ファイルを開いた際に表示されるセキュリティ警告バーの「マクロを有効にする」をクリックしてください
