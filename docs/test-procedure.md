# autoSalesAggre 動作確認手順書

---

## 0. 確認前の準備

### 0-1. Excel Trust Center 設定
1. Excel を起動 → **ファイル → オプション → トラストセンター → トラストセンターの設定**
2. **マクロの設定** → 「すべてのマクロを有効にする」または「署名済みマクロを除くすべてのマクロを無効にする」
3. **「VBA プロジェクト オブジェクト モデルへのアクセスを信頼する」** にチェック ← これがないとワークブック生成が失敗する

### 0-2. ワークブック生成
```
cscript setup\create_workbook.vbs
```
**確認ポイント：**
- コンソールに `Done. Open autoSalesAggre.xlsm in Excel and enable macros.` が表示される
- `autoSalesAggre.xlsm` が生成されている
- エラー `800A0035`（ファイル未発見）や `800A03EC`（VBAプロジェクトアクセス拒否）が出ていない

### 0-3. シート構成確認
`autoSalesAggre.xlsm` を開き、以下の4シートが存在することを確認：

| 順番 | シート名 | 確認内容 |
|------|----------|----------|
| 1 | main | 「実行ログ」ヘッダー、「ファイルを読み込む」ボタン |
| 2 | Config | 製品マスタ/口銭マスタ/ヘッダー名寄せ/部署リスト/SharePoint連携 の5セクション |
| 3 | all | 11列ヘッダー行（客先名〜ソースファイル）、「SharePointへアップロード」ボタン |
| 4 | 集計 | フィルタ欄(A1:B3)、集計ヘッダー(5行目)、グラフ作成・SharePointボタン |

---

## 1. Config シート確認

### 1-1. 製品マスタ（A列・B列）

| 確認内容 | 期待値 |
|----------|--------|
| A1 "製品マスタ"（太字） | ○ |
| A2 "製品コード"、B2 "製品名"（太字） | ○ |
| A3=P001、B3=製品A（サンプルデータ） | ○ |
| A4=P002、B4=製品B（サンプルデータ） | ○ |

### 1-2. 口銭マスタ（D列・E列）

| 確認内容 | 期待値 |
|----------|--------|
| D3=直販、E3=10 | ○ |
| D4=代理店、E4=5 | ○ |

### 1-3. ヘッダー名寄せ（G列・H列）
G3〜G10 に正規名（客先名/製品コード/…）が8行分入っており、H列にエイリアスが入っていることを確認。

### 1-4. SharePoint連携欄（L列・M列）
- L1="SharePoint連携"（太字）
- L2="PowerAutomate URL"（太字）
- M2 は空欄（後で URL を入力する欄）

### 1-5. Power Automate URL の入力
M2 に Power Automate の HTTP 要求トリガー URL を貼り付ける。  
**この手順をスキップするとアップロードボタン押下時にエラーメッセージが出ることを後続で確認する。**

## 2. TSVファイル読み込み確認

### 2-1. テスト用 TSV ファイルの準備
リポジトリ付属の `test_sales.txt` を使用（または下記を `.tsv` で保存）：

```
客先名	製品コード	売上金額	製品単価	売上数量	売上発生日	売上種別	部署
得意先X社	P001	1000000	5000	200	2024/01/15	直販	営業部
得意先Y社	P001	500000	5000	100	2024/02/20	代理店	東京支店
得意先Z社	P002	300000	3000	100	2024/03/10	直販	営業部
```

**ヘッダー名寄せのテスト**として、意図的に列名を変えた第2ファイルも用意する：
```
得意先コード	品番	金額	単価	数量	日付	取引区分	部門
得意先A社	P001	200000	5000	40	2024/04/01	直販	大阪支店
```

### 2-2. 読み込み実行
1. **main シート** の「ファイルを読み込む」ボタンをクリック
2. ファイル選択ダイアログで両ファイルを複数選択（Ctrl+クリック）
3. 「開く」

### 2-3. main シート（ログ）確認

| 確認ポイント | 期待値 |
|--------------|--------|
| 「===== 処理開始 =====」から「===== 処理完了 =====」が出力される | ○ |
| 「X件のファイルを読み込みました」の件数が選択ファイル数と一致 | ○ |
| 「allシート構築完了」「部署リスト更新完了」「集計完了」が順に出る | ○ |
| `[エラー]` 行が出ていない | ○ |

### 2-4. all シート確認

| 確認ポイント | 期待値 |
|--------------|--------|
| 1行目：11列のヘッダーが存在 | 客先名/製品コード/売上金額/製品単価/売上数量/売上発生日/売上種別/部署/製品名/部署取り分/ソースファイル |
| 2行目以降にデータが展開されている | ○ |
| I列（製品名）が「[未登録]」でない | P001→製品A、P002→製品B になっている |
| J列（部署取り分）が計算されている | 売上金額×口銭率/100 の数値 |
| K列（ソースファイル）にファイル名（拡張子なし）が入っている | ○ |
| 先頭ゼロを持つ製品コードが数値変換されていない | テストコード `00123` → 文字列 `00123` のまま |
| 異なるヘッダー名のファイルも正規化されて取り込まれている | 「得意先コード」→客先名列に入っている |

**NG パターン（要確認）：**
- 製品コードが未登録の場合：ログに `警告: 製品コード未登録 [XXX]` が出て、製品名列に `[未登録]` が入る
- 売上種別が未登録の場合：ログに `警告: 売上種別未登録 [XXX]` が出て、部署取り分が `0` になる

### 2-5. シート構成確認
読み込んだファイル数分のシートが `all` と `集計` の間に挿入されていることを確認。

### 2-6. 2回目読み込みの上書き確認
同じファイルを再度読み込んだとき、all シートのデータが**重複せず**クリアされて再構築されることを確認。

## 3. 集計機能確認

### 3-1. フィルタなし（全件集計）
1. 集計シートの B1（部署選択）が「全部署」であることを確認
2. B2・B3（開始日・終了日）が空欄であることを確認
3. データが読み込まれている状態で B1 を一度クリックして再選択（Worksheet_Change を発火させる）

**確認ポイント：**

| 確認ポイント | 期待値 |
|--------------|--------|
| 6行目以降に集計行が展開されている | ○ |
| 製品グループ行（太字・灰背景）の直下に客先行（字下げ）が並ぶ | ○ |
| 製品グループ行の合計 = 配下の客先行合計の総和 | 手計算で検証 |
| 最終行「総合計」行が存在し、全行の合計と一致 | ○ |
| 部署取り分（D列）= 売上金額 × 口銭率/100 の合計 | 手計算で検証 |

### 3-2. 部署フィルタ
1. B1 のドロップダウンを開き、「営業部」を選択
2. 集計テーブルが再描画されることを確認

**確認ポイント：**
- 営業部のデータだけが集計されている（all シートで部署=営業部の行だけ合算）
- 他部署のデータが含まれていない

### 3-3. 日付フィルタ
1. B2 に `2024/01/01`、B3 に `2024/02/28` を入力
2. 集計が 1〜2月データのみになることを確認

**エラーケース確認：**
- B2 に `abc` など無効な日付を入力 → 「開始日の形式が正しくありません」メッセージが出て処理が中断される
- 中断後に B1 のドロップダウン等を変更しても、イベントが正常に動作することを確認（EnableEvents が正しく復元されている）

### 3-4. all シートが空の場合
- all シートのデータを手動で削除 → B1 変更 → 集計テーブルが**クリアされず**に現状維持されることを確認（`ClearAggrTable` は `lastRow < 2` のとき呼ばれない設計）

## 4. グラフ作成確認

### 4-1. 正常系
1. 集計データがある状態で集計シートの「**グラフ作成**」ボタンをクリック
2. 「グラフを作成しました。(X製品)」メッセージを確認

**確認ポイント：**

| 確認ポイント | 期待値 |
|--------------|--------|
| グラフが集計テーブルの**直下**に挿入されている | ○ |
| X軸に製品名（グループ親行のみ）が並んでいる | 客先名・総合計は含まれない |
| 青の棒 = 売上金額合計、オレンジの棒 = 口銭総額 の2系列 | ○ |
| グラフタイトルにフィルタ条件が反映されている | 例: `製品別売上集計　[営業部]　(2024/01/01 ～ 2024/02/28)` |
| 凡例が下部に表示されている | ○ |
| 金額棒の上部にデータラベル（`#,##0` 形式）が表示されている | ○ |
| 縦軸ラベルが「金額（円）」 | ○ |

### 4-2. 再実行で上書き更新
1. 部署フィルタを変更して集計を更新
2. 再度「グラフ作成」ボタンをクリック
3. グラフが1つだけになっていること（重複生成されない）を確認
4. タイトル・データが新しいフィルタ条件に更新されている

### 4-3. データなしのケース
1. all シートを空にした状態でボタン押下
2. 「集計データがありません」メッセージが出て処理が中断される

## 5. SharePoint アップロード確認

### 5-1. URL 未設定エラー（正常なエラー処理確認）
1. Config シート M2 を**空欄のまま**にする
2. 集計シート・all シート両方のアップロードボタンを押下
3. 「ConfigシートのM2にPowerAutomate URLが設定されていません」のエラーメッセージが出ることを確認

### 5-2. 集計シートのアップロード（正常系）

**前提：** Power Automate フローを事前に作成し、HTTP 要求トリガーの URL を Config M2 に設定済みであること。

1. 集計シートで部署・期間を任意に設定して集計を表示させる
2. 「**SharePointへアップロード**」ボタンをクリック
3. 「SharePointへのアップロードが完了しました。」メッセージを確認

**Power Automate 側の受信データ検証：**

```json
{
  "dept": "全部署",
  "fromDate": "2024/01/01",
  "toDate": "2024/02/28",
  "uploadedAt": "2026/04/04 12:00:00",
  "rows": [
    {"name": "製品A",       "amount": 1500000, "qty": 300, "margin": 125000},
    {"name": "　　得意先X社","amount": 1000000, "qty": 200, "margin": 100000},
    {"name": "総合計",      "amount": 1800000, "qty": 400, "margin": 140000}
  ]
}
```

| 確認ポイント | 期待値 |
|--------------|--------|
| `dept`/`fromDate`/`toDate` がフィルタ条件と一致 | ○ |
| `rows` に製品グループ行・客先行・総合計行がすべて含まれる | ○ |
| `amount`/`qty`/`margin` が数値型（文字列でない） | ○ |
| 小数点がピリオド（ロケール非依存） | ○ |
| main シートのログに「SharePointアップロード完了 (HTTP 202)」が記録されている | ○ |

### 5-3. all シートのアップロード（正常系）
1. all シートの「**SharePointへアップロード**」ボタンをクリック
2. 「allシートのSharePointへのアップロードが完了しました。X件のデータを送信しました。」を確認

**受信 JSON 検証：**

```json
{
  "uploadedAt": "2026/04/04 12:00:00",
  "rows": [
    {
      "client":   "得意先X社", "prodCode": "P001",
      "amount":   1000000,     "unitPrice": 5000,
      "qty":      200,         "date":      "2024/01/15",
      "saleType": "直販",      "dept":      "営業部",
      "prodName": "製品A",     "margin":    100000,
      "source":   "test_sales"
    }
  ]
}
```

| 確認ポイント | 期待値 |
|--------------|--------|
| `rows` の件数が all シートのデータ行数と一致 | ○ |
| 全11フィールドが正しいキー名で含まれる | ○ |
| `prodName` が製品マスタから引いた名前になっている | ○ |
| `margin` が計算済みの数値になっている | ○ |

### 5-4. HTTP エラーのケース
Power Automate URL を意図的に壊した値（例: `https://invalid.example.com`）にして送信し、エラーメッセージとログへの記録を確認。

## 6. 複合テスト（E2E）

以下の手順を1通り実行し、全機能が連携して動作することを確認：

1. Config シートにマスタデータと Power Automate URL を設定
2. 異なるヘッダー形式の TSV ファイルを2〜3本用意
3. main シートの「ファイルを読み込む」で全ファイルを一括読み込み
4. all シートでデータ件数・製品名・部署取り分を確認
5. 集計シートで部署と日付を絞り込んで集計を確認
6. グラフ作成 → 絞り込み結果がグラフに反映されているか確認
7. 集計シートからアップロード → Power Automate で受信確認
8. all シートからアップロード → Power Automate で受信確認

---

## 7. 再ビルド確認

`autoSalesAggre.xlsm` を削除後に `cscript setup\create_workbook.vbs` を再実行し、上記 0-3 のシート構成確認まで問題なく完了することを確認する。
